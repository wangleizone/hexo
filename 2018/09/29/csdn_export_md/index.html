使用hexo搭建githb博客

2018/09/29

没有github博客的程序员不是一个好司机，拥有专属github pages域名的博客，完全自定义theme，有没有很强的归属感呢，话不多说开整。

安装node及git环境

http://nodejs.cn/ 

https://git-scm.com/downloads

windows下，都是一直点下一步装的。

安装hexo

全局安装hexo-cli

npm install -g hexo-cli

在本地选一个任意目录init一个hexo工程

hexo i blog //init的缩写 blog是项目名
cd blog //切换到站点根目录
hexo g //generetor的缩写
hexo s //server的缩写 启动hexo工程

如果你的hexo命令一直无法识别，你就需要把刚刚安装的hexo-cli的bin目录配置到系统的环境变量中；

hexo默认端口为4000，打开浏览器输入localhost:4000即可看到hexo默认博客

安装自己喜欢的hexo主题

官网上用户分享的主题

https://hexo.io/themes/

我采用的是从官网中找的Aath

https://github.com/lewis-geek/hexo-theme-Aath

在刚刚创建的工程目录blog/下执行

git clone -b master https://github.com/lewis-geek/hexo-theme-Aath.git themes/aath

安装此主题依赖

npm install –save hexo-renderer-sass

配置主题

hexo 有两个配置文件，一个在 hexo 项目的根目录，另一个在主题文件夹的根目录，文件名都是均为 _config.yml 。

将根目录的_config.yml中的theme配置为aath

About 页面
如果需要生成 about 页面，需要在 blog/source 目录下创建一个 about 目录，并在 about 目录下创建一个 index.md 文件，为该文件添加 layout: about 属性。此时就可在 <your_url>/about 路径访问到 about 页面了。

Tags 页面
如果需要生成 tags 页面，需要在blog/ source 目录下创建一个 tags 目录，并在 tags 目录下创建一个 index.md 文件，为该文件添加 layout: tags 属性。此时就可在 <your_url>/tags 路径访问到 tags 页面了。

重新生成hexo运行代码

hexo clean
hexo g
hexo s

此时再放完localhost:4000主题已经变为aath的，切about和tags页面均正常

创建并配置github仓库

Github 有两种形式的 page：
个人或组织的 page：只能存在一个，master 分支，地址为 xxx.github.io
项目 page：每个项目可以生成一个，gh-pages 分支，地址为 xxx.github.io/projectname
https://help.github.com/articles/user-organization-and-project-pages/

官方参考 https://help.github.com/articles/user-organization-and-project-pages/

创建仓库的名称必须与“你的用户名.github.io”，否则你的博客的访问域名将会是
xxx.github.io/projectname

进入repository的settings中配置pages主页
选择master branch 然后点击save

配置git的ssh链接方式到本地

本地配置global的username和email

git config –global user.name “wangleizone” //Github里面的账号
git config –global user.email “928752160@qq.com” //Github注册的邮箱

在本地生成SSH密钥并配置到Github中

cd ~/.ssh //打开本机的user目录
ssh-keygen -t rsa -C “928752160@qq.com” //执行命令后要求输入的项，直接回车就好

到本机的用户目录C:\Users{username}.ssh找到id_rsa.pub，记事本打开，拷贝内容
粘贴到git中

上传hexo代码

配置blog/._config.yml的git repo提交地址

安装部署需要的依赖

npm install –save hexo-deployer-git

本地仓库目录执行

hexo deploy
即可成功部署到github

输入github page域名体验一下吧
https://wangleizone.github.io
